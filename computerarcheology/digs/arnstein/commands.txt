Dig notes:
7/2/2026. I'm digging into Pyramid and HauntedHouse to see which came first and how the command list
evolved.

PY included a "after every command" function call (right before "Get Object Info" section). In HH1, the call
was removed. In HH2, the call is in place, but the destination is a RET -- like it wasn't cleaned. Pretty
conclusive HH is stripped down PY.

The "container" function is in both PY and HH though it is never used in HH. Evidence that PY came first.

The command functions are in the same order in PY and HH. I would expect new functions to appear at the
end of the list, but they are sprinkled through. Evidence that HH was stripped down from PY.

The general commands are in the same order for PY and HH, but PY has more. I would expect new functions to
appear at the end of the list. Evidence that HH was stripped down from PY.

I see "container" support in both PY and HH.  

In HH, the parse-user-input prints "I DON'T UNDERSTAND". But in PY, a section was added to run a list of 4 messages.

In both games, the gramar-type is at 467D. Coincidence? Looks like it.

In HH, the parse-user-input prints "THERE's NOT ONE HERE", but PY adds a longer sequence to print "I SEE NO <obj> HERE."

PY44F6, the length of word matching was increased from 4 to 6

PY453E, the parse-user-input copies the unknown verb to storage for later error reporting. Same with input noun.

PY4585 - new CopyWord function to copy words into print buffers for message

PY45AC -- added two bytes of storage that are unused by anything
